# <a name="workflow-is-not-starting"></a>Η ροή εργασίας δεν ξεκινά

- Οι ροές εργασίας του SharePoint 2010 και του SharePoint 2013 δεν ξεκινούν.

    - Εάν η ροή εργασίας σας δεν ξεκινά, μπορεί να υπάρχει ένα πρόβλημα προσωρινής υπηρεσίας όπου οι χρήστες ενδέχεται να εμφανίσουν περιοδικές καθυστερήσεις με την πρόοδο της ροής εργασίας. Επιλέξτε τον [πίνακα εργαλείων εύρυθμης λειτουργίας υπηρεσίας](https:/admin.microsoft.com/AdminPortal/Home#/servicehealth) για να δείτε εάν ο οργανισμός σας επηρεάζεται.

    - Εάν έχουν περάσει περισσότερες από 24 ώρες από την πρώτη φορά που είδατε αυτό το πρόβλημα, καταγράψτε ένα δελτίο υποστήριξης. Σε πολλές περιπτώσεις, ήδη εργαζόμαστε σε μια λύση. Παρακαλούμε δώστε μας τουλάχιστον 24 ώρες για να ολοκληρώσετε μια λύση.

- Οι ροές εργασίας του SharePoint 2010 καθυστέρησαν στην έναρξη.

    - Αυτό συμβαίνει εάν η ροή εργασίας ενεργοποιείται σε μεγάλες δέσμες. (για παράδειγμα, όταν προστίθενται ταυτόχρονα πολλά στοιχεία).

    - Οι ροές εργασίας δεν έχουν σχεδιαστεί για να εκτελούνται σε πραγματικό χρόνο, επομένως μια καθυστέρηση είναι η συμπεριφορά κατά τη σχεδίαση.

   -  Εάν η ροή εργασίας είναι σύνθετη γλώσσα σήμανσης επεκτάσιμων αντικειμένων (XMOL), η μεταγλώττιση μπορεί να είναι αργή. Ανατρέξτε σε [αυτό το](https://support.microsoft.com//kb/3043697) άρθρο.

    - Θα πρέπει να απλοποιήσετε τη ροή εργασίας ή να την επανασχεδιάσετε χρησιμοποιώντας τον τύπο πλατφόρμας ροής εργασίας του Microsoft SharePoint 2013.

    - Εάν το ιστορικό ροής εργασίας σας έχει μεγαλώσει, μπορεί να θέλετε να εκκαθαρίσετε τα στοιχεία ή να δημιουργήσετε μια νέα λίστα ιστορικού.

        Περισσότερες πληροφορίες: [Εκκαθάριση ιστορικού ροής εργασίας](https://blogs.technet.microsoft.com/marj/2015/08/07/sharepoint-2010-workflows-best-practice-purge-workflow-history-list-items/)
